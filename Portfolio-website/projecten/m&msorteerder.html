Inleiding

Er zijn twee klassen betrokken bij dit project: de klas 5TWE (Technologische Wetenschappen Engineering) en de klas 5ICW (Informatica Communicatie Technieken). De klas 5TWE gebruikt een kleurensensor om de kleuren van de m&m’s te identificeren, terwijl wij in 5ICW een Raspberry Pi Camera Module 3 gebruiken. Om deze camera te benutten, hebben we twee benaderingen onderzocht: een oplossing met computer vision en OpenCV, of een geavanceerdere aanpak met een machine learning model.

Hoewel OpenCV eenvoudig en snel kan werken, heeft het enkele nadelen: het is gevoelig voor lichtomstandigheden en de foutenmarge kan te groot zijn. Bovendien is het lastig om m&m’s volledig te isoleren van de achtergrond, wat tot verkeerde kleurdetecties kan leiden. Daarom focus ik me op het implementeren van een machine learning model, waarmee we deze problemen kunnen omzeilen en een robuustere oplossing kunnen bouwen.


Mijn Werk


Stap 1: Het probleem definiëren

Het doel is om met behulp van een Raspberry Pi en een camera-module een model te creëren dat m&m’s kan herkennen en hun kleur correct kan identificeren. Hierbij is een machine learning model de voorkeursmethode vanwege de hogere nauwkeurigheid en betere aanpassing aan variabele omstandigheden. Het model moet foutloos kleuren identificeren, zelfs in complexe situaties zoals schaduwen en wisselende lichtomstandigheden.


Stap 2: Data verzamelen

Een goed machine learning model begint met een hoogwaardige dataset. Voor dit project is een dataset nodig die:

  • Alle kleuren m&m’s omvat.
  • Verschillende lichtomstandigheden en hoeken bevat.
  • Genoeg variatie biedt om het model robuust te maken.

Omdat er geen kant-en-klare dataset bestaat voor dit specifieke probleem, overweeg ik om een database zelf te creëren. Dit kan door foto’s te maken van m&m’s onder verschillende omstandigheden, en deze foto’s handmatig of automatisch te labelen (bijvoorbeeld met LabelImg of OpenCV).


Stap 3: Het model ontwerpen

Voor het model lijkt een Convolutional Neural Network (CNN) de meest geschikte keuze, aangezien dit type netwerk bijzonder effectief is in beeldherkenning. Voor de implementatie op een Raspberry Pi zijn modellen zoals ResNet en MobileNet geschikt vanwege hun balans tussen prestaties en efficiëntie.

Om te voorkomen dat het model overfitting ontwikkelt, gebruik ik technieken zoals een validatieset en early stopping. Dit helpt om het aantal trainingsrondes (epochs) te beperken en de prestaties te verbeteren.


Stap 4: Model evalueren

Na het trainen wordt het model geëvalueerd met behulp van een confusion matrix. Dit omvat:

  • True Positives (TP): Correct geïdentificeerde kleuren.
  • True Negatives (TN): Correct uitgesloten kleuren.
  • False Positives (FP): Onterecht als een kleur aangeduide pixels.
  • False Negatives (FN): Niet-herkende kleuren.

Stap 5: Optimalisatie

Het model wordt verder verbeterd door extra data toe te voegen, vooral in gevallen waar het fouten maakt. Dit proces herhaal ik totdat het model aan alle eisen voldoet. Met tools zoals TensorFlow kan ik de voortgang van het model nauwkeurig monitoren.
